
FRONTs connect
-> 각 FRONT 로 부터 현재 연결된 CLIENT 수를 받아서 저장
-> 각 FRONT 로 부터 로그온/오프 시그널 받아서 리스트 업데이트
-> 각 FRONT 가 연결이 끊어진 경우 처리

CLIENT Connect
-> FRONT 리스트 돌면서 CLIENT 를 제일 적게 보유하고 있는 FRONT 의 IP, PORT 전달

FRONT - CLIENT Logon
-> FRONT 가 전달하는 세션 제한 등 정보 저장
-> 세션 수 확인해서 오버된 경우 리스트 돌면서 제일 먼저 로그인 한 FRONT 에게 LOGOFF 명령


typedef map<FRONT_NAME, list<CLIENT_INFO>> MAP_FRONT
typedef map<CLIENT ID, list<FRONT INFO>> MAP_CLIENT
